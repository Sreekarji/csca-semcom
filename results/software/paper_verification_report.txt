CRITICAL GAPS AFFECTING KEY RESULTS
================================================================================

GAP 1: HDM ISR ADVANTAGE (Paper +42.19%, Our +15-27%)
  Status: PARTIAL
  Impact: Affects absolute numbers, NOT qualitative trend
  Cause: 
    - Our channel simulation uses simplified formulas (numpy), not full 3GPP
    - Environment doesn't differentiate well between intelligent and random allocation
    - Baselines trained 2000 episodes (paper may use more)
    - HDM trained 500 episodes (best checkpoint ep500)
  Evidence: Log shows HDM ISR=0.294 vs SAC=0.258 at n=5 (+14%)
  Affects: ISR improvement claim (+42.19%) — absolute number lower
  Qualitative: YES — HDM consistently beats baselines at training config (n=5)
  What's needed:
    - More realistic channel sim with actual resource competition
    - Longer training (2000+ episodes)
    - Environment where intelligent allocation actually matters

GAP 2: DELAY (Paper -33.40%, Our: HDM delay >> baselines)
  Status: PARTIAL
  Impact: HDM has HIGHER delay than baselines (opposite of paper)
  Cause:
    - HDM generates low bandwidth allocations from learned policy
    - Channel sim gives enormous delays at low bandwidth (data_size/rate)
    - Baselines use random/uniform actions that happen to get reasonable BW
  Evidence: Log shows HDM delay=1281s vs Static=64s (20x worse)
  Affects: Delay reduction claim (-33.40%) — direction is reversed
  Qualitative: NO — this is a fundamental issue
  What's needed:
    - Fix bandwidth normalization in HDM output
    - Ensure HDM learns to allocate minimum viable bandwidth
    - Or adjust delay formula to be less sensitive to BW allocation

GAP 3: PSNR FOR IMAGES (Paper Eq. 37)
  Status: PARTIAL
  Impact: PSNR implemented in RL-ASC baseline but not in main CSCA pipeline
  Cause:
    - No image reconstruction pipeline in main code
    - image_eval.py exists but SD load test just passed (not yet run end-to-end)
  What's needed:
    - Run image_eval.py end-to-end with SD 2.1
    - Integrate PSNR into multimodal evaluation

GAP 4: MULTIMODAL SEMCOM PERFORMANCE (Fig 6)
  Status: PARTIAL
  Impact: Cannot reproduce Fig 6 exactly
  Cause:
    - Text: DeepSC baseline works but uses standalone implementation
    - Audio: DASC uses synthetic mel features (not real VoxCeleb audio)
    - Images: RL-ASC uses synthetic 32x32 images (not real Google Landmarks)
  What's needed:
    - Real VoxCeleb audio files (need HF auth)

GAP 5: RELAY SELECTION (Eq. 15)
  Status: PARTIAL

  What's needed:

    - Implement full Eq. 15 computation (not just cosine approximation)
