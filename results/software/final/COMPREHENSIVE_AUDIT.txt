1.1 code/evaluation/cscqi.py
----------------------------------------------------------------------
PURPOSE: Core evaluation metrics — CSCQI, ISR, semantic accuracy, compression ratio.

KEY FUNCTIONS:
  get_semantic_model() -> SentenceTransformer
    - Lazy-loads all-MiniLM-L6-v2 from D:\MP2\all-MiniLM-L6-v2 on CPU.
    - Global singleton (_semantic_model).

  compute_cscqi(tau_S, vartheta_S, tau_S_int, vartheta_S_int) -> float
    - CSCQI metric (paper Eq. 17).
    - Formula: W_TAU * (TAU_MAX - tau_S)/(TAU_MAX - tau_S_int) + W_VARTHETA * (VARTHETA_MAX - vartheta_S)/(VARTHETA_MAX - vartheta_S_int)
    - Returns weighted sum of delay score + quality score.

  compute_isr(tasks: list) -> float
    - Intent Satisfaction Rate.
    - Counts tasks where tau_S <= tau_S_int AND vartheta_S <= vartheta_S_int.
    - Returns satisfied/total.

  compute_semantic_accuracy(sent_text, recv_text) -> float
    - Sentence-level cosine similarity via SentenceTransformer.
    - Returns value in [0, 1].

  compute_compression_ratio(original_text, simplified_text, original_bits, transmitted_bits) -> float
    - If bits provided: transmitted_bits / original_bits.
    - If text provided: simplified_words / original_words.

  is_intent_satisfied(tau_S, vartheta_S, tau_S_int, vartheta_S_int) -> bool
    - Returns tau_S <= tau_S_int AND vartheta_S <= vartheta_S_int.

HARDCODED CONSTANTS:
  TAU_MAX = 10.0        (maximum delay in seconds)
  VARTHETA_MAX = 1.0    (maximum distortion, normalized)
  W_TAU = 0.5           (delay weight, equal per paper)
  W_VARTHETA = 0.5      (quality weight)
  Epsilon 1e-8 in denominators.

DESIGN DECISIONS:
  - SentenceTransformer loaded on CPU to avoid VRAM contention.
  - Model path hardcoded to D:\MP2\all-MiniLM-L6-v2 (offline mode).

1.2 code/evaluation/shaped_reward.py
----------------------------------------------------------------------
PURPOSE: Shaped reward function for HDM training (per-task delay/quality signals + BW-urgency alignment).

KEY FUNCTIONS:
  compute_shaped_reward(tasks, intent_vector, actions) -> float
    - Returns float in [-3.0, 3.0].
    - Components:
      1. Per-task delay score (0.7 weight): +1.0 if met, -2.0*overshoot if missed, clipped [-3, 2]
      2. Per-task quality score (0.3 weight): +1.0 if met, -1.0*overshoot if missed, clipped [-2, 1]
      3. Intent satisfaction bonus: +0.5 flat when both met
      4. BW-urgency alignment: 0.3 * corrcoef(bandwidth, urgency), -0.1 if uniform BW
    - Urgency = 1.0 / max(tau_int, 0.1)
    - Base reward = mean(task_rewards)
    - Final clipped to [-3.0, 3.0]

  compute_isr_reward(tasks) -> float
    - ISR for logging (not training). Same as compute_isr in cscqi.py.

HARDCODED CONSTANTS:
  Weights: 0.7 delay, 0.3 quality, 0.5 bonus, 0.3 alignment
  Clip ranges: delay [-3, 2], quality [-2, 1], overall [-3, 3]
  Urgency floor: 0.1
  Uniform BW penalty: -0.1

DESIGN DECISIONS:
  - FIX (2026-07-08): Stronger per-task delay signal + BW-urgency alignment bonus.
  - Rewards HDM for giving more bandwidth to more urgent tasks.

    index 2: 16QAM, order=16,  rate=0.50, max_ber=1e-3
    index 3: 64QAM, order=64,  rate=0.67, max_ber=1e-4
    index 4: 256QAM,order=256, rate=0.75, max_ber=1e-5
  BERT max_length=512
  BERT model: "bert-base-uncased"
  Batch size: 64
  eta default: 0.60
  epsilon_weight default: 0.1
  BER numerical floor: 1e-6

DESIGN DECISIONS:
  - FIX (2026-07-08): Batched BERT encoding. Original called _get_embedding() once per

  - Early-exit on first successful removal level preserved (Algorithm 1).

1.11 code/hdm/ddpm_policy.py
----------------------------------------------------------------------
PURPOSE: DDPM policy network (MLP denoiser) + Critic network for HDM.

KEY CLASSES:
  MLPDenoiser(nn.Module):
    __init__(action_dim=45, graph_emb_dim=256, task_emb_dim=256, hidden_dim=256, n_denoising_steps=6, n_tasks=5)
    forward(a_n, n, graph_emb, message_embs) -> predicted_noise
    - Split architecture: task_bw_head (per-task BW) + global_head (relay + MCS)
    - Time embedding: Embedding(n_steps+1, 256)
    - task_bw_head: Linear(task_emb+hidden, 256) -> SiLU -> Linear(256, 1)
    - global_head: Linear(action+graph+hidden, 256) -> SiLU -> Linear(256, 256) -> SiLU -> Linear(256, relay_mcs_dim)

  HDMPolicy(nn.Module):
    __init__(action_dim=45, graph_emb_dim=256, n_denoising_steps=6, beta_min=0.01, beta_max=0.5, n_tasks=5)
    _build_schedule()  — precomputes betas, alphas, alphas_cumprod (Eq. 31-32)
    collect_trajectory(graph_emb, message_embs, batch_size) -> (a_0, total_log_prob)
    reverse_diffusion(graph_emb, message_embs, batch_size) -> a_0
    forward(graph_emb, message_embs) -> a_0
    parse_action(a_0, n_tasks, n_relays, n_mcs) -> {bandwidth, relay, mcs}

  CriticNetwork(nn.Module):
    __init__(state_dim=256, action_dim=45, hidden_dim=256)
    forward(graph_emb, action) -> value

HARDCODED CONSTANTS:
  action_dim: 45 (5 tasks + 5*5 relay + 5*3 mcs)
  graph_emb_dim: 256
  n_denoising_steps: 6
  beta_min: 0.01
  beta_max: 0.5
  hidden_dim: 256
  Activation: SiLU (denoiser), ReLU (critic)

EQUATIONS (DDPM noise schedule):
  Eq. 31-32: beta_n = 1 - exp(-(beta_min/N) - (2n-1)/(2N^2) * (beta_max - beta_min))
  Eq. 28-30: reverse diffusion
    coeff = beta_n / sqrt(1 - alpha_bar_n)
    mu_theta = (1/sqrt(alpha_n)) * (a_n - coeff * predicted_noise)
    beta_tilde_n = (1 - alpha_cumprod_prev) / (1 - alpha_cumprod_n) * beta_n  (Eq. 29)
  Final: a_0 = sigmoid(a_n)

DESIGN DECISIONS:
  - MLP denoiser (not UNet) — paper explicitly states MLP for faster inference.
  - Split architecture: task-specific BW head + global relay/MCS head.
  - DDPO-SF: only final step (n=1) carries gradient; intermediate steps detached.
  - Log prob: -0.5 * mean((noise - predicted_noise)^2)

2.4 results/software/final/failure_research_report.txt

TASK 1 — DDPM ACTOR LOSS ROOT CAUSE:
  Current: log_pi = -tanh(MSE(noise, predicted_noise)/10)  — WRONG
  Correct (DDPO, Black et al. 2023): log p(x_{t-1}|x_t, c) = -||x_{t-1} - mu_theta||^2 / (2*sigma_t^2)
  Key insight: reward is SAME for all denoising steps; only log-probs change.
  Our errors: (1) noise MSE not reconstruction likelihood, (2) single step not all T,
  (3) tanh normalization destroys gradient, (4) single-step advantage.

TASK 2 — ALTERNATIVE CHANNELS:
  DeepMIMO: pip installable, 3GPP TR 38.901, CPU-based. RECOMMENDED.
  Sionna: NVIDIA, TensorFlow, GPU. Too heavy.

TASK 3 — COMPRESSION RATIO:
  Paper's 73% = MSS on LAM descriptions (longer, more redundant).
  Our 93% = MSS on SST sentences (already concise).
