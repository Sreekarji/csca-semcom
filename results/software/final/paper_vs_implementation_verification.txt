TASK 2 — ISR DEFINITION VERIFICATION
================================================================================

PAPER'S ISR DEFINITION (Section VI.B, page 31):
  "When the cosine similarity between the sent data and the data restored
  by the receiver is greater than 80% (for text and audio) or the PSNR
  is greater than 22 dB (for image), we consider the SemCom to be accurate."

Intent is satisfied when ALL of:
  1. Semantic accuracy > 80% (cosine similarity for text/audio, PSNR>22dB for image)
  2. Delay <= delay_intent (C1 constraint)
  3. Quality <= quality_intent (C1 constraint)

OUR ISR (compute_isr in cscqi.py):
  Checks ONLY:
  1. tau_s <= tau_int (delay constraint)
  2. vartheta_s <= vartheta_int (quality/distortion constraint)

  Does NOT check:
  3. Semantic accuracy > 80% (requires actual DeepSC encoder)

IMPACT: MAJOR. Our ISR counts tasks as "satisfied" based on channel
metrics only. The paper requires actual semantic accuracy from DeepSC
encoding. This means our ISR is INFLATED — we count tasks as satisfied
that the paper would NOT count (because semantic accuracy might be <80%).

However, our ISR is still much lower than the paper's (9.47% vs ~75%).
This means the channel metrics alone are already too restrictive.
Adding the semantic accuracy check would make our ISR even lower.

CONCLUSION: The ISR definition mismatch is NOT the cause of our low ISR.
The cause is elsewhere (environment calibration, training, or evaluation).

TASK 5 — DDPM POLICY VERIFICATION
================================================================================

NOISE SCHEDULE:
  Paper Eq. 31-32: Variational Posterior scheduler
  Ours: VarianceSchedule with beta_min=0.01, beta_max=0.5, cosine schedule
  Status: MATCH (implements variational posterior)

REVERSE DIFFUSION:
  Paper Eq. 28-30: standard DDPM reverse process
  Ours: MLPPolicy implements Eq. 28-30 with MLP denoiser
  Status: MATCH

ACTOR LOSS:
  Paper Eq. 33: L_pi = E[log pi(at|st) * (RW_acc - V(st))]
  Ours: actor_loss = -critic(graph_emb, action).mean()
  Status: MISMATCH — we use DDPG-style loss, not the paper's
  log-probability weighted advantage. The paper uses log pi(at|st)
  which requires computing the log-probability of the action under
  the DDPM policy. We skip the log-prob term.

IMPACT: HIGH. The actor loss formulation is critical for training.
Using -Q(s,a) (DDPG) vs log*pi(a|s)*(A) (policy gradient) can
produce very different training dynamics. The paper's formulation
explicitly weights the advantage by the log-probability, which
provides better gradient signal for the DDPM policy.

CRITIC LOSS:
  Paper Eq. 35: Lv = E[(R_acc - V(st))^2]
  Ours: MSE loss between target and predicted value
  Status: MATCH

REWARD:
  Paper Eq. 34: RW_acc = sum(gamma^i * RW_{t+i}) — cumulative discounted
  Ours: single-step CSCQI reward (not cumulative discounted return)
  Status: MISMATCH — we use single-step reward, not cumulative

IMPACT: MODERATE. The paper uses cumulative discounted return (Eq. 34)
which accounts for future rewards. Our single-step reward only considers
the current step's CSCQI. This reduces the quality of the value estimate
and may slow convergence.

CURRICULUM LEARNING:
  Paper: not mentioned
  Ours: Easy -> Medium -> Hard curriculum
  Status: NOT IN PAPER (our addition)

  Ours: not measured (no DeepSC integration in evaluation)
  GAP: NOT MEASURED

MOST LIKELY CAUSES OF ISR GAP (ranked by impact):

1. TASK COUNT MISMATCH (HIGH IMPACT):
   Paper tests 2-20 tasks PER CSCA (10-100 total tasks across 5 CSCAs).
   We test 2-20 TOTAL tasks. At n=5 CSCAs with 2 tasks total, each CSCA
   handles ~0.4 tasks. The paper's "20 tasks" means 20 tasks per CSCA
   = 100 total tasks. This fundamentally changes the resource pressure.

2. ACTOR LOSS FORMULATION (HIGH IMPACT):
   Paper uses log-probability weighted advantage (Eq. 33).
   We use DDPG-style -Q(s,a). This can cause the DDPM policy to
   converge to suboptimal solutions.

3. SINGLE-STEP vs CUMULATIVE REWARD (MODERATE IMPACT):
   Paper uses cumulative discounted return (Eq. 34).
   We use single-step CSCQI. This reduces value estimation quality.

4. DELAY INTENT RANGE (MODERATE IMPACT):
   Paper: 50-1000ms (0.05-1.0s) full range
   Our HP: 0.3-0.5s (narrow range)
   Our standard: 0.05-1.0s (matches paper)
   The narrow HP range may not provide enough diversity for learning.

5. TRAINING EPISODES (LOW-MODERATE IMPACT):
   Paper: 100 episodes (but likely with different batch/episode structure)
   Ours: 500 episodes
   The paper may define "episode" differently (e.g., one episode = full
   epoch over all tasks, not a single environment step).

6. mmWave MIMO NOT IMPLEMENTED (LOW IMPACT):
   Paper uses mmWave MIMO channel model. Our simplified model may
   produce different SINR distributions, but the overall structure matches.

7. RELAY SELECTION SIMPLIFIED (LOW IMPACT):
   Paper uses semantic mutual information (Eq. 15) for relay selection.
   We use cosine similarity approximation. This affects distortion
   calculations but is unlikely to cause an 8x ISR gap.

OVERALL VERDICT
================================================================================

SINGLE MOST LIKELY CAUSE OF 8x ISR GAP:

The task count mismatch is the most likely primary cause. The paper tests
with 10-100 total tasks (2-20 per CSCA × 5 CSCAs), while we test with
2-20 total tasks. At low task counts, resources are abundant and all
methods achieve high ISR. The paper's HDM advantage emerges at high task
counts where resource competition is intense.

Our scale comparison (n=2 to n=20) shows HDM advantage growing with
task count (+350% at n=20 vs SAC), which is consistent with the paper's
finding. But we're testing at much lower absolute task counts.

SECONDARY CAUSE: Actor loss formulation.
Paper uses log-probability weighted advantage (Eq. 33), which is the
standard policy gradient for generative models. We use DDPG-style -Q(s,a),
which may not properly train the DDPM policy.

IS IT FIXABLE WITHOUT THE PAPER'S EXACT TRAINING CODE?

YES, partially:
1. Fix task counts: test with 10-100 total tasks (2-20 per CSCA × 5 CSCAs)
2. Fix actor loss: implement log-probability weighted advantage (Eq. 33)
3. Fix reward: implement cumulative discounted return (Eq. 34)
4. These three fixes together could close most of the gap.

WHAT WOULD BE NEEDED TO CLOSE THE GAP FULLY:

1. Implement the paper's exact actor loss (Eq. 33) with log-probability
2. Implement cumulative discounted return (Eq. 34)
3. Test with 10-100 total tasks matching paper's Fig 9a x-axis
4. Train with the correct environment (10MHz BW, 3-5MB data, paper intents)
5. Optional: integrate DeepSC for semantic accuracy in ISR computation
