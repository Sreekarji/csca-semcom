================================================================================

ISSUE 5.1 — train_batch_episode() uses DDPG, not Eq. 33 [CRITICAL]
  File: code/hdm/hdm_trainer.py, lines 455-458
  The main training loop (train() line 490) calls train_batch_episode().
  train_batch_episode() uses: actor_loss = -q_value.mean() (DDPG)
  train_episode() has Eq. 33: actor_loss = -(log_prob * advantage).mean()
  But train_episode() is NEVER called by train().
  Impact: The paper's Eq. 33 actor loss is never used in actual training.
  Fix: Replace DDPG loss in train_batch_episode() with Eq. 33.

ISSUE 5.2 — HAN gradients detached in actor path [CRITICAL]
  File: code/hdm/hdm_trainer.py, line 457
  q_value = self.critic(graph_embs.detach(), action_new)
  The .detach() prevents gradients from flowing back to HAN through actor.
  HAN only gets gradients through critic (line 448), which learns value,
  not action quality.
  Impact: HAN cannot learn to produce better graph embeddings for actions.
  Fix: Remove .detach() from graph_embs in actor path.

ISSUE 5.3 — Replay buffer push in train_batch_episode()
  Line 433: self.replay_buffer.push(graph_emb, action, reward_value, next_graph_emb)
  But the replay buffer is never sampled in train_batch_episode()!
  It's only sampled in train_episode() (line 298).
  Since train_episode() is never called, the replay buffer is useless.
  Severity: LOW — replay buffer is unused

ISSUE 5.4 — Curriculum schedule
  Phase 1 (Easy): episodes 1-200
  Phase 2 (Medium): episodes 200-500
  Phase 3 (Hard): DISABLED (returns "medium" for ep > 500)
  Impact: Training never reaches hard difficulty
  Severity: MODERATE

ISSUE 5.5 — Reward signal variation
  CSCQI reward: clipped to [-5, 5]
  In lenient environment: most tasks satisfied → CSCQI ~20-30 → clipped to 5
  In hard environment: some tasks fail → CSCQI varies more
  Impact: In lenient env, reward signal is flat (all 5.0) — no gradient
  Severity: HIGH — actor can't learn if reward doesn't vary

CATEGORY 8 — ROOT CAUSE ANALYSIS OF ISR GAP
================================================================================

Current ISR: ~94% (after environment calibration fix)
Paper ISR: ~90%
Gap: We're now ABOVE the paper's target!

BUT: Our environment is much more lenient than paper's.
Paper uses 50-1000ms delay intents; we use 0.1-5.0s.
Paper uses 90-100% quality intents; we use 30-100%.

If we used paper's exact parameters:
  - Delay intents 50-1000ms with our channel model → most tasks would fail
  - Quality intents 90-100% with our distortion model → most tasks would fail

TOP 5 CAUSES OF REMAINING ISSUES:

1. Environment too lenient (HIGH impact)
   Our data sizes (0.1-1MB) and bandwidth (100MHz) make tasks trivially easy.
   Paper's environment is much tighter.
   Fix: Reduce bandwidth to 10-20MHz, increase data sizes to 1-10MB.

2. train_batch_episode() doesn't use Eq. 33 (HIGH impact)
   The paper's actor loss (log_pi * advantage) is never used.
   DDPG loss (-Q) is used instead.
   Fix: Replace DDPG with Eq. 33 in train_batch_episode().

3. HAN gradients detached (HIGH impact)
   HAN can't learn to produce better graph embeddings for actions.
   Fix: Remove .detach() from graph_embs in actor path.

4. Delay intents too wide (MODERATE impact)
   0.1-5.0s vs paper's 0.05-1.0s
   Fix: Narrow delay intent range to 0.05-1.0s.

5. Quality intents too low (MODERATE impact)
   0.3-1.0 vs paper's 0.9-1.0
   Fix: Set quality_intent = 0.9-1.0 for all tasks.
